# Security Commands in Linux

In this article, we'll go over some of the most important security-related commands in Linux. These commands help you monitor activity on your machine and maintain system security — whether you're using Linux as a personal device, a server, or in a work environment.

## 1. ss 📝 — Monitor connections and network activity

The `ss` command is one of the most powerful tools for checking open connections and network activity.

**Why is it important?**

- Shows open ports
- Displays active connections
- Helps you detect suspicious network activity
- Faster and more accurate than netstat

**Examples:**

```bash
ss                # list all connections
ss -a             # list all listening + non-listening ports
ss -t             # show only TCP connections
ss -lnt           # list listening ports only with details
```

## 2. who 👀 — See who is currently logged in

The `who` command displays all users currently logged into the system.

**Why is it important?**

- Helps you check if someone accessed your system without permission
- Useful on shared servers or multi-user environments

```bash
who
```

## 3. last 💻 — Review login history

The `last` command shows the login and logout history for all users.

**Why is it important?**

- Helps monitor security activity
- Identifies unusual login attempts
- Shows timestamps and the source IP address used

```bash
last
```

## 4. passwd 🔒 — Change and manage user passwords

The `passwd` command is simple but essential for security.

**Tips:**

- Always use a strong password
- Change it periodically
- Disable unused accounts whenever possible

```bash
sudo passwd Abdullah
```

## 5. ufw 🔥 — Simple and effective firewall

UFW stands for Uncomplicated Firewall, a very easy-to-use tool for managing firewall rules.

**Why is it important?**

- Blocks unnecessary ports
- Protects your machine from unauthorized access
- Great for small and medium servers

**Examples:**

```bash
sudo ufw enable        # enable the firewall
sudo ufw status        # check firewall status
sudo ufw allow 22      # allow traffic on a specific port (e.g., SSH)
sudo ufw deny 80       # block traffic on a specific port
sudo ufw delete allow 22
```
