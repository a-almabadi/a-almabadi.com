# System Monitoring Commands in Linux

This week's Linux article is all about commands you can use for system monitoring.

These tools help you understand what's happening inside your machine — from CPU usage to memory, disk activity, processes, and network traffic.

Check them out below:

## 1. uptime

The `uptime` command shows how long the system has been running.

It also displays:

- Current time
- System load averages
- Number of logged-in users

**Example:**

```bash
uptime
```

## 2. free

Use `free` to check memory usage on your system.

It shows:

- Total RAM
- Used RAM
- Free memory
- Swap usage

**Example:**

```bash
free -h
```

## 3. ps

`ps` displays the currently running processes.

**Examples:**

```bash
ps
ps aux    # show all running processes
```

## 4. tcpdump

`tcpdump` is a powerful tool used to analyze network traffic.

**Example:**

```bash
sudo tcpdump
```

## 5. w

The `w` command shows who is logged in and what they are doing.

**Example:**

```bash
w
```

## 6. htop

`htop` is an interactive process viewer — similar to `top` but more advanced and user-friendly.

It shows:

- CPU usage
- Memory usage
- Running processes in real time

**Example:**

```bash
htop
```

## 7. iostat

`iostat` monitors I/O performance of storage devices.

**Useful for:**

- Checking disk bottlenecks
- Viewing read/write speeds
- Monitoring disk activity

**Example:**

```bash
iostat
```

## 8. lsof

`lsof` stands for List Open Files.

It shows:

- Which files are open
- Which process opened them
- Ports used by processes

**Example:**

```bash
sudo lsof
```

## 9. pidstat

`pidstat` monitors individual processes and their resource usage.

**Great for:**

- Troubleshooting
- Tracking CPU spikes
- Analyzing misbehaving applications

**Example:**

```bash
pidstat
```

## 10. top

`top` displays the most active processes in real time.

It helps you see:

- CPU usage
- Memory usage
- Load average
- Resource-heavy tasks

**Example:**

```bash
top
```
