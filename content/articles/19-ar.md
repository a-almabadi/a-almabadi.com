# أساسيات الشبكات: شرح مبسط لقوائم التحكم ACL

## ما هي الـ ACL؟

ACL هي مجموعة من القواعد تُستخدم للتحكم في مرور الترافيك داخل الشبكة، ويتم كتابتها داخل الراوتر أو السويتش. وتعمل بناءً على عدة معايير، مثل:

- عنوان الـ IP (المصدر أو الوجهة)
- نوع البروتوكول (TCP, UDP, ICMP)
- رقم المنفذ مثل 80 أو 443
- نوع الخدمة أو أي خصائص أخرى تعتمد على نوع الـ ACL

## مهام الـ ACL

تقوم قوائم التحكم بالوظائف التالية:

- السماح أو منع أنواع معينة من الترافيك
- حماية الشبكة من الوصول غير المصرح به
- تقليل استهلاك موارد الشبكة
- التحكم في الاتصال عبر المنافذ و خطوط VTY الخاصة بـ Telnet و SSH

## أنواع قوائم التحكم ACL

تنقسم الـ ACL إلى نوعين رئيسيين:

- **NUMBERED ACL (مرقمة)**
- **NAMED ACL (مسماة)**

وكل نوع منهما يحتوي على نوعين:

- **Standard ACL (قياسية)**
- **Extended ACL (موسعة)**

### أولاً: Standard ACL

- تتحكم فقط في عنوان المصدر
- لا تستطيع التحكم في الوجهة أو البروتوكول أو أرقام المنافذ
- تطبق عادة بالقرب من الوجهة

**مثال:**

```
R1(config)# access-list 55 permit 192.168.10.0 0.0.0.255
```

هذه ACL قياسية رقمها 55 تسمح للشبكة 192.168.10.0 فقط.

### ثانياً: Extended ACL

توفر تحكماً أكبر وتشمل:

- المصدر والوجهة
- البروتوكول
- رقم المنفذ

وتطبق عادة بالقرب من المصدر.

![Extended ACL Placement](https://i.postimg.cc/Kz78zL28/5.jpg)

**مثال:**

```
R1(config)# access-list 103 permit tcp 192.168.10.0 0.0.0.255 any eq 443
```

هذه ACL موسعة تسمح لترافيك HTTPS الخاص بالشبكة 192.168.10.0.

## مصطلحات مهمة داخل ACL

| الكلمة | المعنى |
|---------|--------|
| `HOST` | يستخدم لجهاز واحد فقط |
| `ANY` | جميع العناوين |

## ملاحظات مهمة

- جميع الـ ACL القياسية تحتوي بشكل ضمني على:
  ```
  deny any
  ```

- وجميع الـ ACL الموسعة تحتوي ضمنياً على:
  ```
  deny ip any any
  ```

لذلك يجب دائماً إضافة سطر سماح Permit مناسب عند الحاجة.

## أوامر ACL الأساسية

- `permit` = السماح
- `deny` = المنع
- `remark` = تعليق توضيحي لا يؤثر على التنفيذ

## أمثلة إضافية على NUMBERED ACL

### Standard:

```
R1(config)# access-list 55 permit 192.168.10.0 0.0.0.255
```

### Extended:

```
R1(config)# access-list 103 permit tcp 192.168.10.0 0.0.0.255 any eq 443
```

## أمثلة على NAMED ACL

### Standard:

```
R1(config)# ip access-list standard ABC
R1(config-std-nacl)# deny 192.168.10.0 0.0.0.255
R1(config-std-nacl)# permit any
```

### Extended:

```
R1(config)# ip access-list extended FFF
R1(config-ext-nacl)# deny tcp host 192.168.10.2 any eq www
R1(config-ext-nacl)# deny tcp host 192.168.10.3 any eq http
R1(config-ext-nacl)# deny tcp host 192.168.10.4 any eq 80
R1(config-ext-nacl)# deny tcp host 192.168.10.5 host 192.168.20.7 eq ftp
R1(config-ext-nacl)# permit ip any any
```

## تطبيق الـ ACL على المنافذ

```
R1(config)# interface G0/0
R1(config-if)# ip access-group NAME or NUMBER in | out
```

- `in` = الترافيك الداخل
- `out` = الترافيك الخارج

## تطبيق ACL على خطوط VTY (استخدامها مع Telnet و SSH)

```
R1(config)# line vty 0 15
R1(config-line)# ip access-class NAME or NUMBER in | out
```

## عرض قوائم التحكم

| الأمر | الوظيفة |
|-------|----------|
| `show access-lists` | عرض جميع القوائم |
| `show ip interface G0/0` | معرفة القوائم المطبقة على المنفذ |

## أفضل مكان لتطبيق كل نوع

| النوع | المكان الأنسب |
|-------|----------------|
| Standard ACL | عند الوجهة |
| Extended ACL | عند المصدر |

![ACL Placement Guide](https://i.postimg.cc/kgDGcBxT/4.jpg)

**السبب:**

- Standard بسيطة وقد تمنع حركة مهمة لو طُبّقت عند المصدر، بينما Extended دقيقة وتقلل الحمل عند تطبيقها من بداية الترافيك.
