# Networking Basics: A Simple Guide to ACLs

## What Is an ACL?

An ACL (Access Control List) is a set of rules configured on a router or switch to control the flow of traffic based on specific criteria, such as:

- Source or destination IP address
- Protocol type (TCP, UDP, ICMP)
- Port number (80, 443, 21, etc.)
- Service type
- And other parameters depending on the ACL type

ACLs are essential for securing the network, controlling access, and reducing unnecessary traffic.

## What Do ACLs Do?

ACLs are used to:

- Permit or deny certain types of traffic
- Protect networks from unauthorized access or attacks
- Optimize network performance by reducing unwanted traffic
- Secure remote access (via VTY lines)
- Apply rules on router interfaces to control inbound or outbound traffic

## Types of ACLs

ACLs come in two main categories:

- **Numbered ACLs**
- **Named ACLs**

Each category has two types:

- **Standard ACLs**
- **Extended ACLs**

### 1. Standard ACL

Standard ACLs filter traffic based only on the source IP address.

- They cannot filter based on destination, protocol, or port.
- **Best Placement:** Near the destination
  (to avoid unintentionally blocking important traffic)

**Example**

```
R1(config)# access-list 55 permit 192.168.10.0 0.0.0.255
```

This standard ACL permits all traffic coming from the 192.168.10.0/24 network.

### 2. Extended ACL

Extended ACLs offer more advanced control. They filter based on:

- Source and destination IP
- Protocol (TCP, UDP, ICMP, etc.)
- Port number (HTTP, HTTPS, FTP...)

- **Best Placement:** Near the source, to prevent unwanted traffic early.

![Extended ACL Placement](https://i.postimg.cc/Kz78zL28/5.jpg)

**Example**

```
R1(config)# access-list 103 permit tcp 192.168.10.0 0.0.0.255 any eq 443
```

This rule allows TCP traffic from the 192.168.10.0/24 network to any destination on port 443 (HTTPS).

## Important Keywords

| Keyword | Meaning |
|---------|---------|
| `host` | Specifies a single IP address (instead of using wildcard 0.0.0.0) |
| `any` | Matches all addresses (instead of wildcard 255.255.255.255) |

## Hidden Notes You Should Know

- Every standard IPv4 ACL ends with an implicit:
  ```
  deny any
  ```

- Every extended IPv4 ACL ends with an implicit:
  ```
  deny ip any any
  ```

This means any traffic not explicitly permitted will be denied by default.

## Core ACL Commands

- `permit` → Allow traffic
- `deny` → Block traffic
- `remark` → Add a descriptive comment to the ACL

## Examples of ACL Types

### Numbered Standard ACL

```
R1(config)# access-list 55 permit 192.168.10.0 0.0.0.255
```

### Named Standard ACL

```
R1(config)# ip access-list standard ABC
R1(config-std-nacl)# deny 192.168.10.0 0.0.0.255
R1(config-std-nacl)# permit any
```

### Named Extended ACL

```
R1(config)# ip access-list extended FFF
R1(config-ext-nacl)# deny tcp host 192.168.10.2 any eq www
R1(config-ext-nacl)# deny tcp host 192.168.10.3 any eq http
R1(config-ext-nacl)# deny tcp host 192.168.10.4 any eq 80
R1(config-ext-nacl)# deny tcp host 192.168.10.5 host 192.168.20.7 eq ftp
R1(config-ext-nacl)# permit ip any any
```

ACL "FFF" blocks specific traffic and permits everything else.

## Applying ACLs to Router Interfaces

```
R1(config)# interface G0/0
R1(config-if)# ip access-group NAME_or_NUMBER in | out
```

- `in` = filter traffic entering the interface
- `out` = filter traffic leaving the interface

## Applying ACLs to VTY (Remote Access)

Used to protect Telnet/SSH connections.

```
R1(config)# line vty 0 15
R1(config-if)# ip access-class NAME_or_NUMBER in | out
```

## Viewing ACLs

| Command | Description |
|---------|-------------|
| `show access-lists` | Displays all ACLs |
| `show ip interface G0/0` | Shows ACLs applied on an interface |

## Placement Guide

| ACL Type | Best Placement |
|----------|----------------|
| Standard | Near the destination |
| Extended | Near the source |

![ACL Placement Guide](https://i.postimg.cc/kgDGcBxT/4.jpg)

**Reason:**

- Standard ACLs are less specific—placing them near the source may block important traffic unintentionally.
- Extended ACLs are more precise, so applying them at the source cleans unwanted traffic early.
