# Networking Basics: A Detailed Guide to VLANs

This article provides a complete and detailed explanation, from basic concepts to practical implementation, including real-world examples used by companies.

## What is a VLAN?

**VLAN = Virtual Local Area Network**

A VLAN is a logical network created within a switch.

Instead of having the entire network in one broadcast domain, VLANs allow you to segment it into multiple domains without purchasing additional switches.

## Main Purpose of VLANs

VLANs help segment network devices based on:

- Department (IT, HR, Finance)
- Security level
- Type of data
- Required services
- User type

## Why Use VLANs?

| Benefit | Simple Explanation |
|---------|---------------------|
| Traffic Isolation | Each department has its own network and cannot access another VLAN directly |
| Increased Security | Reduces the risk of sniffing or internal attacks |
| Reduced Broadcast | Minimizes unnecessary broadcast traffic, improving performance |
| Lower Network Load | Segmented network keeps traffic organized and efficient |
| Traffic Control | Define what enters and exits each VLAN |
| Easier Management & Scalability | Adding a new VLAN is easier than installing a new switch |

## Important Terms to Understand

### 1. Broadcast Domain

All devices that receive broadcast traffic from each other.

- **Without VLAN** → The entire company is one domain
- **With VLAN** → Each VLAN is its own domain

### 2. Access Port

- Port assigned to a single VLAN
- Typically used for end-user devices

### 3. Trunk Port

- Port that carries multiple VLANs between switches or between a switch and a router
- Uses 802.1Q tagging so the switch knows which VLAN each packet belongs to

### 4. Native VLAN

- VLAN that passes through the trunk without a tag
- Usually VLAN 1 by default, but changing it is recommended for security reasons

## Types of VLANs in Detail

### 1. Default VLAN

- Usually VLAN 1
- All ports are assigned to it by default

### 2. Data VLAN

- For regular users
- **Example:**
  - VLAN 10 → Finance
  - VLAN 20 → HR
  - VLAN 30 → IT

### 3. Voice VLAN

- Dedicated to VoIP phones
- Provides higher call quality and network priority (QoS)

### 4. Management VLAN

- Dedicated to managing switches using:
  - SSH, Telnet, SNMP, HTTP/HTTPS
- Typically VLAN 99 or VLAN 100

### 5. Native VLAN

- Untagged VLAN used between switches in trunk mode

## How to Create a VLAN on a Switch

### 1. Create VLAN

```
Switch(config)# vlan 10
Switch(config-vlan)# name IT
```

### 2. Assign VLAN to an Access Port

```
Switch(config)# interface f0/1
Switch(config-if)# switchport mode access
Switch(config-if)# switchport access vlan 10
```

## Inter-VLAN Routing

Devices in the same VLAN can communicate naturally.

To allow communication between VLANs (e.g., VLAN 10 → VLAN 20), you need:

- A router (Router-on-a-Stick)
- Or a Layer 3 switch

### Example: Router-on-a-Stick

```
interface g0/0
 no shutdown

interface g0/0.10
 encapsulation dot1Q 10
 ip address 192.168.10.1 255.255.255.0

interface g0/0.20
 encapsulation dot1Q 20
 ip address 192.168.20.1 255.255.255.0
```

## Real-World Example

| Department | VLAN | Network |
|------------|------|---------|
| Finance | 10 | 192.168.10.0 |
| HR | 20 | 192.168.20.0 |
| IT | 30 | 192.168.30.0 |

- Each department is isolated
- IT can access all departments
- HR and Finance cannot communicate directly

**Implementation Steps:**

1. Create VLAN for each department
2. Apply ACLs to control traffic
3. Use Inter-VLAN routing for necessary connections

## Common VLAN Issues

### VLAN Mismatch

- Switch 1 trunk sends VLAN 20, Switch 2 does not accept it

### Different Native VLANs

- Switch A → Native VLAN 1
- Switch B → Native VLAN 30
- Can cause security and communication issues

### VLAN Not Allowed on Trunk

- Port is trunk but the VLAN is not permitted
- **Solution:**
  ```
  switchport trunk allowed vlan 10,20,30
  ```

## VLAN Security Best Practices

- Avoid using VLAN 1
- Create a dedicated Management VLAN
- Shut down unused ports
  ```
  interface range f0/10-24
   shutdown
  ```
- Use Port Security
- Apply ACLs to VLANs
- Only allow necessary VLANs on trunks

## Enhancing Your Network with VLANs

- Isolate IoT devices
- Create a Guest VLAN for visitors
- Assign VLANs for cameras
- Assign VLANs for servers
- Improve voice and video quality
- Secure the Management VLAN

## Conclusion

VLANs are essential for any modern network. They provide:

- Organization
- Security
- Control
- Scalability
- High performance

And all this is possible without extra hardware, just using the switch itself.
